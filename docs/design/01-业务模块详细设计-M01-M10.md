#### 1.2.2 Service
**`ReviewServiceImpl.submit(userId, req)`**

伪代码：
```java
@Transactional
public ReviewResultVO submit(Long userId, ReviewSubmitReq req) {
    // 1. 行数校验（前置）
    int lineCount = countLines(req.getCode());
    if (lineCount > 500) throw new CodeTooLargeException(500);

    // 2. 创建审查主表（PENDING）
    Review review = new Review();
    review.setUserId(userId);
    review.setCodeContent(req.getCode());
    review.setFileName(req.getFileName());
    review.setLineCount(lineCount);
    review.setStatus("PENDING");
    review.setReviewTime(LocalDateTime.now());
    reviewRepository.insert(review);

    try {
        long start = System.currentTimeMillis();
        // 3. 拉取启用的规则
        List<Rule> rules = ruleService.listEnabled();
        // 4. 引擎审查
        ReviewContext ctx = reviewEngine.review(req.getCode(), req.getFileName(), rules);
        if (ctx.getCompilationUnit() == null && ctx.getIssues().isEmpty()) {
            review.setStatus("FAILED");
            review.setErrorMsg("代码语法错误: " + ctx.getParseError());
            reviewRepository.updateById(review);
            throw new CodeParseException(ctx.getParseError());
        }
        // 5. 落库
        List<Issue> issues = persistIssues(review.getId(), ctx.getIssues());
        // 6. 异步 AI 增强
        enrichWithAi(issues);
        // 7. 更新状态
        review.setStatus("COMPLETED");
        review.setIssueCount(issues.size());
        review.setCostMs(System.currentTimeMillis() - start);
        reviewRepository.updateById(review);
        return buildResultVO(review, issues);
    } catch (BizException e) { throw e; }
    catch (Exception e) {
        review.setStatus("FAILED");
        review.setErrorMsg(e.getMessage());
        reviewRepository.updateById(review);
        throw new BizException("审查失败: " + e.getMessage());
    }
}
```

#### 1.2.3 行数计算工具
**`ReviewServiceImpl.countLines(String code) → int`**

```java
private int countLines(String code) {
    if (code == null) return 0;
    if (!code.isEmpty() && code.charAt(0) == '\uFEFF') code = code.substring(1); // 去 BOM
    if (code.isEmpty()) return 0;
    int n = 1;
    for (int i = 0; i < code.length(); i++) {
        if (code.charAt(i) == '\n') n++;
    }
    if (code.endsWith("\n")) n--;
    return Math.max(n, 1);
}
```

## M02 规范检查引擎

### 2.1 目标
基于 JavaParser 解析出的 `CompilationUnit` AST 节点，遍历检查类名、方法名、常量名、格式、方法长度等规范违规。

### 2.2 后端实现

#### 2.2.1 入口
**`engine/executor/RuleExecutor.java`** —— 所有检查器统一接口

```java
public interface RuleExecutor {
    void execute(ReviewContext context, Rule rule);
    String category();   // STYLE / DEFECT / SECURITY
    String patternType(); // AST / REGEX
}
```

#### 2.2.2 内置检查器（3 个，全部 category=STYLE, patternType=AST）

| Bean 名 | 类 | 规则 code | 检查内容 |
|---------|----|-----------|---------|
| `namingChecker` | `NamingChecker` | `NAMING_CLASS_UPPERCAMEL` | 类名 `^[A-Z][a-zA-Z0-9]*$` |
| ↑ | ↑ | `NAMING_METHOD_LOWERCAMEL` | 方法名 `^[a-z][a-zA-Z0-9]*$` |
| ↑ | ↑ | `NAMING_CONSTANT_UPPER_SNAKE` | 常量名 `^[A-Z][A-Z0-9]*(_[A-Z0-9]+)*$` |
| `formatChecker` | `FormatChecker` | `STYLE_LINE_TOO_LONG` | 单行字符数 > 120 |
| `structureChecker` | `StructureChecker` | `STYLE_METHOD_TOO_LONG` | 方法行数 > 80 |

#### 2.2.3 NamingChecker 关键实现

```java
@Component("namingChecker")
public class NamingChecker implements RuleExecutor {
    private static final Pattern CLASS_UPPER = Pattern.compile("^[A-Z][a-zA-Z0-9]*$");
    private static final Pattern METHOD_LOWER = Pattern.compile("^[a-z][a-zA-Z0-9]*$");
    private static final Pattern CONSTANT_UPPER = Pattern.compile("^[A-Z][A-Z0-9]*(_[A-Z0-9]+)*$");

    public void execute(ReviewContext ctx, Rule rule) {
        CompilationUnit cu = ctx.getCompilationUnit();
        if (cu == null) return;
        switch (rule.getCode()) {
            case "NAMING_CLASS_UPPERCAMEL" -> cu.findAll(ClassOrInterfaceDeclaration.class)
                .forEach(c -> check(CLASS_UPPER, c.getNameAsString(), "类名", c, rule, ctx));
            // ...
        }
    }
}
```

#### 2.2.4 扩展机制
新增规则 = 新建 `xxxChecker implements RuleExecutor` + 标注 `@Component("xxxChecker")` + 数据库 `rule.executor_bean = "xxxChecker"` + 启用即可。**无需修改引擎主控**。

规范类问题在 `IssueCard.vue` 中以蓝色左边框渲染：
- `<el-tag type="primary">规范</el-tag>`
- 折叠面板显示「代码示例」「修复建议」

## M03 缺陷检测引擎

### 3.1 目标
识别 Java 代码中的常见缺陷：空指针、资源泄漏、空 catch、宽 catch。

### 3.2 后端实现

#### 3.2.1 内置检测器（4 个，全部 category=DEFECT, patternType=AST）

| Bean 名 | 类 | 规则 code | 检查策略 |
|---------|----|-----------|---------|
| `nullPointerDetector` | `NullPointerDetector` | `DEFECT_NULL_POINTER` | 启发式：方法调用前未判空 + 变量非字面量赋值 |
| `resourceLeakDetector` | `ResourceLeakDetector` | `DEFECT_RESOURCE_LEAK` | IO 资源类型 new 出来后未在 try-with-resources 也未 close() |
| `emptyCatchDetector` | `EmptyCatchDetector` | `DEFECT_EMPTY_CATCH` | `catch` 块语句数为 0 |
| `broadCatchDetector` | `BroadCatchDetector` | `DEFECT_BROAD_CATCH` | 捕获 `Exception` 或 `Throwable` |

#### 3.2.2 NullPointerDetector 核心算法

```java
public void execute(ReviewContext ctx, Rule rule) {
    cu.accept(new VoidVisitorAdapter<>() {
        @Override
        public void visit(MethodCallExpr mc, Object arg) {
            Optional<Expression> scope = mc.getScope();
            if (scope.isEmpty()) return;
            String varName = extractVarName(scope.get());
            if (varName == null) return;
            if (isProbablyNull(cu, varName, mc.getBegin().get().line)) {
                ctx.getIssues().add(IssueDraft.builder()...build());
            }
        }
    }, null);
}

private boolean isProbablyNull(CompilationUnit cu, String varName, int callLine) {
    // 1) 调用前已 if (varName != null)  -> 安全
    for (IfStmt ifStmt : cu.findAll(IfStmt.class)) {
        if (ifStmt.getCondition().toString().contains(varName + " != null")) {
            int ifStart = ifStmt.getBegin().get().line, ifEnd = ifStmt.getEnd().get().line;
            if (callLine >= ifStart && callLine <= ifEnd) return false;
        }
    }
    // 2) 字面量/字面量表达式赋值 -> 非空
    for (VariableDeclarationExpr vd : cu.findAll(VariableDeclarationExpr.class)) {
        for (VariableDeclarator v : vd.getVariables()) {
            if (!varName.equals(v.getNameAsString())) continue;
            Expression init = v.getInitializer().orElse(null);
            if (init instanceof StringLiteralExpr || init instanceof ObjectCreationExpr || ...) return false;
        }
    }
    // 3) 其余情况视为可疑
    return true;
}
```

> **已知限制**：未做完整数据流分析；只覆盖最常见的场景。生产环境建议升级为基于符号求解的完整 NPE 检测。

#### 3.2.3 ResourceLeakDetector 启发式

资源白名单：
```java
private static final Set<String> RESOURCE_TYPES = Set.of(
    "FileInputStream", "FileOutputStream", "FileReader", "FileWriter",
    "BufferedReader", "BufferedWriter", "InputStreamReader", "OutputStreamWriter",
    "Scanner", "Socket", "ServerSocket", "Connection",
    "Statement", "PreparedStatement", "ResultSet", "ZipFile", ...
);
```

判定：
1. `VariableDeclarationExpr` 初始化表达式是 `ObjectCreationExpr` 且类型在白名单 → 候选
2. 该变量在 `TryStmt.getResources()` 中 → 安全（try-with-resources）
3. 代码中含 `varName.close()` → 安全
4. 其余 → 报告

### 3.3 前端实现
缺陷类在 `IssueCard` 中以橙色左边框 + WARNING/ERROR tag 渲染。

### 3.4 测试要点
参见《详细设计说明书》7.2.1、7.2.2。覆盖：
- NPE：方法调用前未判空 → ERROR
- NPE：方法调用前 `if (s != null)` → 无
- 资源泄漏：try-with-resources → 无
- 资源泄漏：手动 `close()` → 无
- 资源泄漏：未关闭 → ERROR
- 空 catch：1 个 WARNING
- 宽 catch：1 个 WARNING

## M04 安全扫描引擎

### 4.1 目标
优先拦截高危安全漏洞：SQL 注入、硬编码密码、XSS、命令注入。安全类问题严重级别 = `CRITICAL`，报告中置顶。

### 4.2 后端实现

#### 4.2.1 内置扫描器（4 个，全部 category=SECURITY）

| Bean 名 | 类 | 规则 code | patternType |
|---------|----|-----------|-------------|
| `sqlInjectionScanner` | `SqlInjectionScanner` | `SECURITY_SQL_INJECTION` | REGEX |
| `hardcodedSecretScanner` | `HardcodedSecretScanner` | `SECURITY_HARDCODED_SECRET` | REGEX |
| `xssScanner` | `XssScanner` | `SECURITY_XSS` | REGEX |
| `commandInjectionScanner` | `CommandInjectionScanner` | `SECURITY_COMMAND_INJECTION` | REGEX |

> 安全类扫描不依赖 AST，可对**语法错误的代码**继续扫描。

#### 4.2.2 SqlInjectionScanner 正则
```java
Pattern.compile("(?i)(\"(?:SELECT|INSERT|UPDATE|DELETE)[^\"]*\"\\s*\\+\\s*\\w+|\\w+\\s*\\+\\s*\"(?:SELECT|INSERT|UPDATE|DELETE)[^\"]*\")");
```

匹配形式：
- `"SELECT * FROM users WHERE id=" + id`
- `sql + "SELECT * FROM ..."`

#### 4.2.3 HardcodedSecretScanner
```java
// 模式 1: 变量赋值
"(?i)\\b(password|passwd|pwd|api[_-]?key|secret|token|access[_-]?token|app[_-]?secret)\\s*=\\s*\"([^\"]{3,})\""
// 模式 2: 常量声明
"(?i)\\b(?:private|public|protected)?\\s*static\\s+final\\s+String\\s+[A-Z0-9_]*(?:KEY|SECRET|TOKEN|PASSWORD|PWD)\\s*=\\s*\"([^\"]{4,})\""
```

白名单（不报警）：
- 占位符 `${xxx}` / `***` / `xxxx+` / `your_xxx` / `change_me` / `xxx`

#### 4.2.4 执行顺序
在 `ReviewEngine` 中，所有 SECURITY 规则的 `executorBean` 注册到 `executorMap`；执行顺序由 `rules` 列表决定（数据库默认 SECURITY 排在 DEFECT 之前，可在 SQL 初始化时固定 `ORDER BY category DESC`）。

### 4.3 前端实现
`ReportViewer.vue` 中：
- 严重级别统计区：CRITICAL 用红色 `#dc2626`
- 独立「安全风险摘要」区：列出所有 SECURITY 问题，醒目标注
- 顶部「⚠ 安全风险摘要」金色提示卡片

### 4.4 测试要点
- `String sql = "SELECT * FROM users WHERE name='" + name + "'";` → 1 个 CRITICAL
- `private static final String DB_PWD = "root123";` → 1 个 CRITICAL
- `private static final String DB_PWD = "${DB_PWD}";` → 0（占位符）
- `Runtime.getRuntime().exec("ls " + dir);` → 1 个 CRITICAL
- `response.getWriter().write(request.getParameter("name"));` → 1 个 CRITICAL

### 6.1 目标

### 6.2 后端实现

#### 6.2.1 Controller
**`ReportController.export(reviewId, type) → ResponseEntity<byte[]>`**

```java
@GetMapping("/export/{reviewId}")
public ResponseEntity<byte[]> export(@PathVariable Long reviewId, @RequestParam(defaultValue = "pdf") String type) {
    Long uid = SecurityUtil.currentUserId();
    String role = SecurityUtil.currentRole();
    ReviewResultVO result = reviewService.detail(reviewId, uid, role);
    byte[] bytes;
    MediaType mt;
    if ("word".equalsIgnoreCase(type)) {
        bytes = reportService.exportWord(result);
        mt = MediaType.parseMediaType("application/vnd.openxmlformats-officedocument.wordprocessingml.document");
    } else {
        bytes = reportService.exportPdf(result);
        mt = MediaType.APPLICATION_PDF;
    }
    return ResponseEntity.ok().contentType(mt)
        .header("Content-Disposition", "attachment; filename=\"review-" + reviewId + "." + (type.equals("word") ? "docx" : "pdf") + "\"")
        .body(bytes);
}
```

#### 6.2.2 PDF 生成
**`ReportServiceImpl.exportPdf(ReviewResultVO) → byte[]`**

使用 OpenPDF（LGPL 协议 fork of iText 4）：
- 标题：粗体 18 号居中
- 概要：文件名/时间/行数/问题数/耗时
- 严重级别统计：CRITICAL/ERROR/WARNING/SUGGESTION 计数
- 安全风险摘要：仅当存在安全问题
- 问题清单：每条问题独立段，颜色按严重级别
  - CRITICAL → `#dc2626`（红）
  - ERROR → `#ea580c`（橙）
  - WARNING → `#ca8a04`（黄）
  - SUGGESTION → `#2563eb`（蓝）
- AI 解释：斜体紫色

#### 6.2.3 Word 生成
**`ReportServiceImpl.exportWord(ReviewResultVO) → byte[]`**

使用 Apache POI XWPF：
- 标题 + 概要 + 严重级别 + 安全 + 问题清单
- CRITICAL tag 红色
- 修复建议绿色
- AI 解释斜体紫色

### 6.3 前端实现
`ReportViewer.vue` 顶部「导出 PDF」「导出 Word」按钮：
```js
const resp = await service.get('/report/export/' + id, { responseType: 'blob' })
const blob = new Blob([resp.data])
const link = document.createElement('a')
link.href = URL.createObjectURL(blob)
link.download = `review-${id}.${type === 'word' ? 'docx' : 'pdf'}`
link.click()
```

### 6.4 测试要点
- 500 行代码审查后导出 PDF：文件大小 50-200KB，能在 Adobe Reader / Chrome 打开
- 导出 Word：能 MS Word / WPS 打开
- 包含中文字符不乱码：使用系统字体 + UTF-8 编码

#### 7.2.2 热更新机制
**`RuleServiceImpl.refreshCache()`**

```java
public void refreshCache() {
    List<Rule> rules = ruleRepository.selectList(
        new LambdaQueryWrapper<Rule>().eq(Rule::getEnabled, 1));
    memoryCache.set(rules);
    try {
        redisTemplate.delete(CACHE_KEY);
        redisTemplate.opsForValue().set(CACHE_KEY, rules);
        redisTemplate.convertAndSend(CHANNEL_RULE_UPDATED, "refresh");
    } catch (Exception e) {
        log.warn("[RULE] Redis 刷新失败，仅内存缓存生效: {}", e.getMessage());
    }
}
```

**降级策略**：
1. Redis 不可用 → 内存缓存 `AtomicReference<List<Rule>>` 兜底
2. 引擎读取顺序：Redis → 内存
