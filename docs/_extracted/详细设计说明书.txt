1. 引言
1.1 编写目的
本文档在概要设计的基础上，对系统各模块进行详细设计，包括类设计、接口定义、核心算法、数据结构、异常处理等。为编码实现提供精确的技术规范。
1.2 适用范围
本文档适用于系统开发团队的后端开发人员、前端开发人员及测试人员。开发人员应根据本文档进行编码实现，测试人员应根据本文档编写单元测试和集成测试用例。
1.3 设计约定
包命名：com.codeaudit.{module}
类命名：采用大驼峰命名（PascalCase）
方法命名：采用小驼峰命名（camelCase）
常量命名：全部大写+下划线分隔（UPPER_SNAKE_CASE）
数据库表命名：小写+下划线分隔，复数形式
RESTful URL：小写+连字符分隔

5. 核心算法
5.1 代码行数校验算法
在代码提交时，前端和后端分别执行行数校验：
前端：使用Monaco Editor的getModel().getLineCount()获取行数
前端：若行数>500，阻止提交并显示红色警告提示
后端：对接收到的代码文本按\n分割，计算数组长度
后端：若行数>500，返回HTTP 400错误码及提示信息
文件上传：在上传前读取文件内容，预检行数
5.2 AST解析流程
使用JavaParser库进行代码解析，流程如下：
代码预处理：移除BOM头，统一换行符为\n
语法解析：调用JavaParser.parse(code)获取CompilationUnit
解析异常处理：若解析失败（语法错误），记录解析错误并终止后续检查
获取AST节点：从CompilationUnit中获取类型声明、方法声明、字段声明等
节点遍历：使用VoidVisitorAdapter对AST进行深度优先遍历
5.3 问题严重级别排序算法
审查结果中的问题按以下优先级排序：
第一优先级：严重（CRITICAL）—— 安全漏洞类问题
第二优先级：错误（ERROR）—— 逻辑缺陷、空指针等
第三优先级：警告（WARNING）—— 规范违规、异常处理等
第四优先级：建议（SUGGESTION）—— 代码风格、可读性等
同级别内按行号升序排列
5.4 报告生成算法
报告生成采用模板引擎+数据填充的方式：
准备报告模板（HTML模板，支持中文排版）
填充审查概要：时间、文件名、行数、问题统计
按严重级别分组渲染问题列表
安全提示专区独立渲染，使用红色/橙色高亮样式
使用Puppeteer/Playwright将HTML转换为PDF
使用docx库直接生成Word格式报告

8. 部署说明
8.1 环境要求
8.2 部署步骤
传统部署：
安装JDK 17、Node.js 24、MySQL 8.0、Redis 7.0
创建MySQL数据库 code_audit，执行初始化SQL脚本
修改application.yml中的数据库连接和Redis连接配置
执行 mvn clean package 构建后端JAR包
执行 npm run build 构建前端静态资源
配置Nginx反向代理规则，指向后端服务和前端静态目录
启动后端：java -jar code-audit-server.jar
启动Nginx
Docker部署：
确保已安装Docker和Docker Compose
修改docker-compose.yml中的环境变量
执行 docker-compose up -d 启动所有服务
执行 docker-compose ps 验证服务状态
